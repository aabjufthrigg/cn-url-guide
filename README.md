# cn-url-guide

本项目用于归档和发布多个独立的 HTML 页面。这些页面本身不针对任何单个域名或具体网站，仅作为通用内容存档与展示用途。

## 项目简介

cn-url-guide 是一个静态页面集合仓库，旨在以 HTML 格式保存和呈现各类信息。每个页面均为独立文件，不依赖外部数据库或后端服务，可直接通过浏览器打开查看。

## 目录说明

```
.
├── index.html          # 首页/导航页
├── pages/              # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 静态资源文件（CSS、图片、字体等）
│   ├── css/
│   ├── img/
│   └── js/
└── README.md           # 本文件
```

- `index.html`：项目入口页面，提供页面索引或导航。
- `pages/`：所有独立 HTML 页面存放于此，文件名建议使用有意义的英文或拼音。
- `assets/`：页面所需的样式、脚本、图像等资源文件，按类型分目录存放。

## 页面归档说明

- 每个页面均为自包含的 HTML 文件，内容可涵盖文本、表格、链接、图片等常见元素。
- 页面之间无强依赖关系，可单独访问或修改。
- 归档内容应保持简洁、清晰，避免过多外部依赖（如远程 API、外部 CDN 资源），以确保长期可用性。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来补充、修正或更新页面内容。
- 新增页面时，请在 `pages/` 目录下创建新文件，并确保文件名唯一。
- 若涉及资源文件更新，请同步修改 `assets/` 下对应文件。
- 请勿在页面中嵌入任何具体域名、网址、营销承诺或诱导点击内容。
- 本仓库仅作为技术存档与展示空间，不承担其他用途责任。
